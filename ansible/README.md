### Развёртывание Kubernetes через Ansible

Для запуска плейбуков необходимо иметь **два готовых сервера** в облаке Selectel.

---

### 1. Подготовка master-ноды

В этом примере master-нода имеет внешний IP:

```
31.129.63.56
```

Пропишите IP-адрес вашей master-ноды в файле `hosts.ini` в строке `k8smaster`.

Строка `k8work_node` на момент первого запуска должна быть **закомментирована**.

---

### Установка базовых пакетов

Сначала запустите плейбук:

```bash
ansible-playbook -i hosts.ini setup_deb12_k8s.yml
```

Будут установлены предварительные пакеты для работы Kubernetes и создан пользователь.

Пароль пользователя можно изменить — см. комментарии в плейбуке.

---

### Установка Kubernetes на master-ноду

Перед запуском убедитесь, что в файле `all.yml` раскомментирована строка:

```yaml
- debian12-k8s-install
```

После этого выполните:

```bash
ansible-playbook -i hosts.ini all.yml
```

Если всё прошло успешно, в конце вывода вы увидите Kubernetes-поды в статусе:

```
Pending
```

---

# 2. Подготовка worker-ноды

Вернитесь в панель управления Selectel и добавьте внешний IP-адрес для worker-ноды.

В этом примере используется IP:

```
31.129.63.115
```

Пропишите IP вашей worker-ноды в файле `hosts.ini` в строке:

```
k8work_node
```

---

## Подготовка файла `all.yml`

Перед запуском необходимо изменить роли в файле `all.yml`:

```yaml
# - debian12-k8s-install
- debian12-k8s-work-node-setup
```

То есть нужно отключить роль установки master-ноды и подготовить запуск роли `debian12-k8s-work-node-setup`.

---

# 3. Запуск плейбуков для worker-ноды

Сначала выполните базовую настройку:

```bash
ansible-playbook -i hosts.ini setup_deb12_k8s.yml
```

Затем запустите роль для worker-ноды:

```bash
ansible-playbook -i hosts.ini all.yml
```
